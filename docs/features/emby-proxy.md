# Emby 反向代理

MediaTidy 内嵌 Caddy 反向代理引擎，可拦截 Emby 的播放请求并实现 302 重定向直链播放，大幅降低服务器带宽压力。

## 工作原理

```
客户端 → MediaTidy 反代 (Caddy) → Emby Server
                ↓
         拦截视频播放请求
                ↓
         302 重定向到 115 直链
                ↓
         客户端直连 115 下载播放
```

- **普通请求**（页面、API、图片等）直接透传给 Emby
- **播放请求**（`/Videos/*/stream`、`/Videos/*/original` 等）被拦截，由 MediaTidy 生成 115 直链后 302 重定向
- **PlaybackInfo 请求** 被拦截以注入直链信息

## 配置步骤

### 1. 添加 Emby 服务器

进入「Emby 管理」页面，点击「添加服务器」：

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **服务器名称** | `name` | 自定义名称（留空则自动从 Emby API 获取） |
| **Emby 地址** | `embyUrl` | Emby 服务器内网地址，如 `http://192.168.1.100:8096` |
| **API Key** | `embyApiKey` | Emby 管理面板中生成的 API 密钥 |
| **监听端口** | `listenPort` | 反代对外暴露的端口（每台服务器独立端口） |
| **启用** | `enabled` | 开关此服务器的反代 |

::: tip
添加时系统会自动请求 Emby 的 `/System/Info/Public` 接口获取服务器名称和内部 ID，用于后续 Webhook 事件匹配。
:::

### 2. 修改客户端连接

将 Emby 客户端的服务器地址改为 MediaTidy 的反代地址（如 `http://<mediatidy-ip>:<listenPort>`），而非直连 Emby。

### 3. 管理端口

| 配置项 | 字段 | 默认值 | 说明 |
|--------|------|--------|------|
| **管理端口** | `adminPort` | 2019 | MediaTidy 管理面板的 Web 端口 |

管理端口不能与任何 Emby 代理端口冲突。

## 302 直链播放

播放请求被拦截后，MediaTidy 会：

1. 解析请求中的 Emby Item ID
2. 通过 Emby API 获取文件路径
3. 将文件路径转换为 115 下载直链
4. 返回 302 重定向响应，客户端直接从 115 下载播放

**优势：**
- 零服务器带宽消耗
- 播放速度取决于客户端到 115 的网络质量
- 响应头自动设置 `Accept-Ranges: bytes` 支持拖进度条

**转码回落：** 当客户端不支持源文件格式时，请求会透传给 Emby 原生转码处理，确保所有设备均可播放。

## 预缓存

| 配置项 | 字段 | 默认值 | 说明 |
|--------|------|--------|------|
| **启用预缓存** | `preCacheEnabled` | false | 用户浏览详情页时提前解析直链 |
| **预缓存 QPS** | `preCacheQps` | 2 | 每秒预缓存请求数（避免触发 115 限频） |

当用户在 Emby 中浏览电影/剧集详情页时，系统会提前获取该影片的 115 直链并缓存，这样点击播放时可以立即响应，减少等待时间。

---

## 同播复制

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **启用同播复制** | `concurrentCopy` | 当其他用户正在播放同一文件时，自动复制到指定目录获取独立直链 |
| **复制目录** | `concurrentCopyDir` | 复制目标目录名（默认 `最近接收`） |

115 同一文件的下载链接可能在多人同时播放时冲突，同播复制通过为每个用户创建独立副本来解决此问题。

---

## Webhook 联动

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **启用 Webhook** | `webhookEnabled` | 接收 Emby 发送的事件通知 |
| **Webhook 事件** | `webhookEvents` | 订阅的事件类型（留空 = 全部） |
| **通知路由** | `notificationChannelIds` | 事件转发到哪些通知渠道（留空 = 全部启用的渠道） |

### 在 Emby 中配置 Webhook

1. 安装 Emby Webhook 插件
2. 添加 Webhook 地址：`http://<mediatidy-ip>:8080/api/webhook/emby`
3. 选择要监听的事件类型

### 支持的事件

| 事件 | 用途 |
|------|------|
| **Library Changed** | 媒体库变更，触发 STRM 刷新 |
| **Item Added** | 新增媒体项 |
| **Item Removed** | 删除媒体项，可同步清理 STRM |
| **Playback Start** | 播放开始事件 |

## 路径映射

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **路径映射** | `pathMappings` | MediaTidy 本地路径 → Emby 可见路径的映射 |

当 MediaTidy 和 Emby 部署在不同 Docker 容器中时，同一目录的挂载路径可能不同。路径映射用于解决此问题。

```json
[
  {
    "localPrefix": "/app/strm",
    "embyPrefix": "/media/strm"
  }
]
```

系统提供**自动检测**功能：点击「检测路径映射」按钮，系统会对比 STRM/整理输出目录与 Emby 媒体库路径，自动推荐映射配置。

---

## 多服务器支持

支持同时代理多台 Emby 服务器，每台可独立配置端口、预缓存、Webhook 和路径映射。系统自动检测端口冲突。
