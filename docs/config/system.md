# 系统设置

系统设置页面用于配置全局参数，包括 TMDB、代理、日志、媒体扩展名等。所有配置存储在数据库的 `app_config` 表 `key=system` 中。

## TMDB 配置

| 配置项 | 字段 | 默认值 | 说明 |
|--------|------|--------|------|
| **API Key** | `tmdb.apiKey` | 内置默认 | TMDB API 密钥（v3）。系统内置一个默认密钥，也可替换为自己的 |
| **语言** | `tmdb.language` | `zh-CN` | 搜索结果和元数据的显示语言 |

::: warning 重要
TMDB API Key 是自动整理功能的前提条件。系统已内置默认 API Key，一般无需自行配置。如果需要使用自己的 Key，可以在此处填入。
:::

### TMDB 缓存

系统内置 TMDB 持久化缓存，减少重复 API 调用。基本配置在此页面设置：

| 配置项 | 字段 | 默认值 | 说明 |
|--------|------|--------|------|
| **启用缓存** | `enabled` | `true` | 关闭后不再写入新缓存 |
| **最大条目数** | `maxSize` | `5000` | 缓存上限，超出后 LRU 自动淘汰 |

完整的缓存管理功能（搜索、编辑、锁定、刷新、清除等）请参阅 **[TMDB 缓存管理](/features/tmdb-cache)** 专题文档。

## 代理配置

出站代理用于外部 API 调用（如 TMDB、Telegram 通知等）。支持 HTTP 和 SOCKS5 两种代理协议。

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **启用** | `proxy.enabled` | 开关代理 |
| **类型** | `proxy.type` | `http` 或 `socks5` |
| **主机** | `proxy.host` | 代理服务器地址 |
| **端口** | `proxy.port` | 代理服务器端口 |
| **用户名** | `proxy.username` | 代理认证用户名（可选） |
| **密码** | `proxy.password` | 代理认证密码（可选） |
| **代理服务** | `proxy.services` | 指定哪些服务走代理（留空则默认仅 `tmdb`） |

### 代理服务路由

`services` 字段控制哪些外部服务通过代理访问。系统内部区分两类 HTTP 客户端：

- **代理客户端**：`services` 列表中的服务使用此客户端
- **直连客户端**：不在列表中的服务直接连接（如内网 Emby、CD2 等）

如果 `services` 为空且代理已启用，默认仅将 `tmdb` 路由到代理。

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **服务 URL** | `serviceURL` | MediaTidy 对外可访问的 URL（如 `http://192.168.2.197:8080`） |

用于生成 302 重定向回调地址等场景。

## API Key

| 配置项 | 字段 | 说明 |
|--------|------|------|
| **全局 API Key** | `apiKey` | 用于 Webhook 接口鉴权（首次访问时自动生成） |

外部系统调用 MediaTidy 的 Webhook 接口时需要在请求中携带此 Key。

## 日志配置

| 配置项 | 字段 | 默认值 | 说明 |
|--------|------|--------|------|
| **日志级别** | `logLevel` | `info` | `debug` / `info` / `warn` / `error` |

`debug` 级别会输出大量详细信息（包括每次 API 调用、规则匹配过程等），仅在排查问题时建议开启。

可在管理面板的「系统日志」页面实时查看日志输出。

## 媒体扩展名

全局配置识别的媒体文件扩展名。这些配置作为**全局默认值**，整理方案和 STRM 方案中可以覆盖。

扩展名解析优先级：**方案级 > 全局级 > 内置默认**

**视频扩展名（默认）：**
```
.mkv, .mp4, .avi, .ts, .m2ts, .wmv, .flv,
.mov, .webm, .rmvb, .rm, .mpg, .mpeg, .vob, .iso
```

**元数据扩展名（默认）：**
```
.srt, .ass, .ssa, .sub, .idx, .sup,
.nfo, .jpg, .jpeg, .png, .tbn, .txt, .json
```

## 空目录清理

### 空目录判定策略

全局策略定义了判断目录是否为「空」的规则（存储在 `emptyDirPolicy` 配置中）：

| 配置项 | 字段 | 默认值 | 说明 |
|--------|------|--------|------|
| **忽略扩展名** | `ignoreExts` | `.nfo, .jpg, .jpeg, .png, .srt, .ass, .sup, .txt, .url` | 这些扩展名的文件视为不存在 |
| **忽略文件** | `ignoreFiles` | `.DS_Store, Thumbs.db, desktop.ini` | 这些文件名（不区分大小写）视为不存在 |
| **排除目录** | `excludeDirNames` | `@eaDir, #recycle, @Recycle, .@__thumb` | 跳过这些目录（不进入、不删除） |
| **忽略文件大小** | `ignoreMaxSize` | `0` | 小于此大小（字节）的文件视为不存在，`0` = 禁用 |

当目录中所有文件都被上述规则判定为可忽略时，该目录被视为「空目录」，可以被清理。

### 定时清理任务

支持配置定时任务批量清理指定目录中的空文件夹：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **清理目录** | — | 需要扫描清理的目录列表 |
| **排除子目录** | — | 在清理目录中需要跳过的子目录名 |
| **Cron 表达式** | `30 3 * * *` | 定时执行频率（默认每天凌晨 3:30） |
| **启用** | `false` | 是否启用定时清理 |

也可在页面上手动触发一次清理任务。

### 整理后自动清理

在整理方案的配置中可以开启 `cleanEmptyDirs`，整理完成后自动清理源目录中产生的空文件夹。
